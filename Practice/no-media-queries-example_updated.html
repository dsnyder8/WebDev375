Fluid Layout WITHOUT Media Queries

Try This: Resize your browser window to phone size (around 375px wide) and watch what happens to the columns below!

Column 1

This column uses a fluid width of 32%. It will shrink proportionally as the viewport gets smaller.

On desktop: Perfect!

On phone: Too cramped!

Column 2

All three columns maintain their percentage widths no matter how small the screen gets.

The text becomes nearly unreadable on mobile devices.

Column 3

Without media queries, we can't tell these columns to stack vertically on small screens.

They just keep shrinking!

The Problem with Fluid Layouts Alone:

  • Desktop (1200px): Each column is ~384px wide → Readable
  • Tablet (768px): Each column is ~246px wide → Getting tight
  • Phone (375px): Each column is ~120px wide → Unusable!

The columns scale proportionally, but they don't adapt their layout. We need different layouts for different screen sizes, not just smaller versions of the same layout.

How to Test This:

  1. Use the Developer View - Inspect elements
  2. Select "iPhone SE" or "iPhone 12 Pro" from the device dropdown
  3. Watch how cramped the three columns become!

Solution: Media queries let us change the layout at specific breakpoints (e.g., stack columns vertically on screens under 600px).
